In this week’s DevBootcamp content, we focused primarily on manipulating data using arrays, before briefly reviewing hashes. If you're new to all this as well, you'll soon discover that arrays and hashes are far more similar than you may have thought (much moreso than rays and hash...). An array is simply a collection of indexed objects. Those objections can be integers, strings, or even other objects. To access any of the objects in an array, you simply use its index number. Index numbers start at zero, and so the first element of an array is retrieved by using array[0], the second element would be array[1], etc. In the example below, we define an array called array_alphabet of alphabet letters, and use the index numbers to return the letters “d”, “b”, and “c”. 

 
  array_alphabet = [‘a’,’b’,’c’,’d’,’e’,’f’,’g’]
  array_alphabet[3]
  => "d"
  array_alphabet[1]
  => "b"
  array_alphabet[2]
  => "c"
  array_alphabet.values_at(3,2,1)
  => ["d","b","c"]

Arrays are incredibly powerful, and can get far more complex when we nest additional arrays inside the, and want to run complex math functions against their contents. But for the most part, as long as you understand the indexing system used by arrays, you should be able to use them.

So what exactly is a hash then? A hash is very very similar to an array. The only difference is that a hash can have any type of object used as its indexing key. With arrays, our index key are always integers starting zero. Hashes, on the other hand, can take any object (string or integer or float) as its key values. This makes them slightly less efficient from a compute time point of view, but much more flexible. Instead of having to assign your first element to the [0] index position, you can give it a semantically meaningful name, such as “first_name”. Hashes can still use the integer-based indexing if they want, but you would generally use an array in that case, since it would be more efficient. For example, if we want to store data about NHL players, we’ll likely want to be able to retrieve that data based on factors like team, position, etc. And so building an array of NHL players wouldn’t be sufficient. Instead, we’d want to build a hash. The simple hash below creates a new hash called player_info, for the Sharks’ captain Joe Pavelski. We then retrive the player's age using hash notation. 

 
  player_info = {
  name: ‘Joe Pavelski’,
  number: 8,
  height: ‘5”11’,
  weight: 190,
  goals: 2,
  assists: 3,
  team: ‘San Jose Sharks’,
  age: 31
  }

  player_info['age']
  => 31

In a nutshell, that hopefully explains the differences between hashes and arrays. Have questions or comments? Still can't make the on the blog itself, so please drop them in GitHub or Canvas!
